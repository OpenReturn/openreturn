# Contributing to OpenReturn

Thank you for your interest in contributing to OpenReturn. This project is in its early stages and we welcome feedback, ideas and code contributions.

## How to get involved

**Protocol design feedback.** The most valuable contributions right now are on the protocol specification. Open a [discussion](https://github.com/OpenReturn/openreturn/discussions) to share thoughts on the return lifecycle model, exchange flows, carrier abstraction patterns, or agentic interface design.

**Bug reports and feature requests.** Use [GitHub Issues](https://github.com/OpenReturn/openreturn/issues) for concrete bugs or specific feature proposals. Please check existing issues before opening a new one.

**Code contributions.** Fork the repository, create a branch, and submit a pull request. Keep PRs focused on a single change. For larger changes, open an issue or discussion first so we can align on the approach.

**Carrier and platform adapters.** If you have experience with a specific carrier API or e-commerce platform, adapter contributions are especially welcome. See the adapter interface documentation (coming soon) for the contract each adapter must implement.

**Reference portal.** The return portal is built with Next.js. Contributions to the consumer-facing return flow, retailer dashboard, or self-hosting setup are welcome.

**Documentation.** Improvements to docs, tutorials and examples are always appreciated.

## Development setup

Development setup instructions will be added as the codebase takes shape. For now, see the main [README](README.md) for an overview of the project architecture.

## Guidelines

- Write clear commit messages that explain what changed and why.
- Follow the existing code style in the repository.
- Add or update tests for any new functionality.
- Keep pull requests small and focused. One concern per PR.
- Be respectful in all interactions. We follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) code of conduct.
